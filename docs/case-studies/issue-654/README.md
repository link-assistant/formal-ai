# Issue 654: general agentic change planning

## Timeline and evidence

- 2026-07-12: issue 654 identified the `plan_chat_step` recognizer chain as the
  recipe-driven boundary left by PR 601 and issue 558.
- 2026-07-13: the prepared branch and PR 677 were created. The issue and all
  issue/PR comments, reviews, and inline comments were captured under `raw-data/`;
  there was no human review feedback at implementation time.

## Requirements

1. Store a general `.lino` plan shape with a goal, ordered capability steps,
   expected evidence, and verification command.
2. Compose a plan from a formalized requirement instead of requiring an
   `is_*_task` match.
3. Produce the same plan for the same request and seed, and append that plan to
   the tool transcript before execution.
4. Keep every pinned recipe as a regression fixture.
5. Prove at least three unpinned English/Russian requests and one real Agent CLI
   round trip.

## Root cause

`src/agentic_coding/planner.rs::plan_chat_step` ended in `None` after a linear
chain of recognizers. `Progress` also stored a set of completed capabilities,
which made a data plan containing two ordered writes impossible to execute: the
second write was indistinguishable from the first.

## Solution

`general_planner` is a bounded fallback after the stored recipes. It reuses
`formalize_intent` for a stable formalized impulse, extracts a safe relative
file target and explicit payload, and composes plan data with `Write, Write,
Run` steps. The first write persists `.formal-ai/general-change-plan.lino`; the
second performs the requested change; the final command verifies evidence.
Ambiguous targets, traversal, URLs, and invented content are rejected.

The existing recipes remain in front of the fallback, so their behavior is
unchanged. Progress now retains ordered repeated capabilities, while its
existing `done` queries preserve set-like behavior.

## Verification

- `cargo test agentic_general_planner`: per-requirement and whole-task tests.
- Existing `agentic_coding` and `issue_538_agentic` suites: recipe regressions.
- `data/meta/general-change-plan.lino`: specification fixture.
- `test-agent-cli-e2e`: a differently worded task absent from all task constants.
- The committed Agent CLI session and real E2E log under this case study are
  generated by the repository driver/toolchain, not synthesized.
