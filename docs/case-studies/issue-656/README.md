# Issue 656: benchmark-gated promotion

Issue [#656](https://github.com/link-assistant/formal-ai/issues/656) closes the
unsafe gap between proposal-only self-improvement and durable seed knowledge.
PR [#690](https://github.com/link-assistant/formal-ai/pull/690) originally
implemented the event vocabulary and confirmation gate. The maintainer's
2026-07-16 review required the deeper version: fresh executable evidence,
general auto-learning, and the same Formal AI / Agent CLI task path used to
materialize the learned change.

The completed protocol has four trust boundaries:

1. A proposal document is intent only. Runner, floor, result, and CI claims are
   rejected as untrusted input.
2. Formal AI executes the three canonical gates and binds their process output
   to content-addressed evidence. Failure and malformed output block promotion.
3. Accepted changes are coalesced and authored through Formal AI's agentic
   `write_file` path. The path and bytes must match exactly.
4. Apply requires confirmation and a clean Git worktree, creates a local
   `promotion/<run-id>` branch, and never pushes. GitHub checks and human review
   govern the eventual draft PR SHA.

The implementation is split into bounded modules:

- `src/promotion.rs`: protocol types, decisions, event chain, proposal format;
- `src/promotion/gates.rs`: canonical command execution and evidence parsing;
- `src/promotion/materialize.rs`: local branch and Agent-authored exact writes;
- `src/cli_improve.rs`: dry-run/apply/confirmation workflow.

See [requirements.md](requirements.md), [solution-plans.md](solution-plans.md),
and [raw-data/online-research.md](raw-data/online-research.md). Reproducible Agent
CLI and gate logs are generated by `examples/issue-656-promotion/run.sh`.

The preserved 2026-07-16 run promoted one previously unseen modifier after
coding `4/4`, industry `10/10`, and focused unit `9/9` evidence. It created
`promotion/promotion_run_ac401d8d5e0e52ac`; both the in-process Formal AI Agent
session and external Agent CLI wrote the same 79-byte seed payload. The script's
final `cmp` is the byte-reproducibility assertion. Review
`agent-cli-run/session.json`, `agent-cli-run/agent-stream.jsonl`, and
`agent-cli-run/promotion-run.lino` for the two independent execution traces.
