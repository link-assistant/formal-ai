# PR 645 Maintainer Amendment

Source: <https://github.com/link-assistant/formal-ai/pull/645#issuecomment-4937917787>

This later PR conversation comment expanded the acceptance criteria after the
initial raw-data capture. It requires the implementation to:

1. read retained meta-algorithm amendments and apply them to future related
   tasks, backed by end-to-end tests and data;
2. replay the whole problem and verify the expected output before forgetting a
   covered specific;
3. derive candidate tasks from frequent topics, simulate them, preserve failed
   simulations for improvement, and discover recurring structures without
   depending only on English requirement phrases;
4. close issue #494 with real filesystem measurements, actual incoming bytes,
   cached/seed usage, minimal cleanup, persisted CLI and desktop consent, and a
   larger-storage prompt when reclaimable memory cannot satisfy the reserve;
5. drive the gap analysis through Formal AI's Agent CLI, preserve the complete
   session, and survey reusable components and mature libraries;
6. run dreaming in the core runtime as well as Electron, at lowest practical
   priority, only while idle, and yield to foreground work; and
7. use memory-links terminology consistently and load multilingual learning
   cues from data rather than hardcoding English-only logic.

The implementation requirements derived from this amendment are recorded as
`R540-19` through `R540-28` in `../requirements.md` and `R413` through `R422`
in the repository-level `REQUIREMENTS.md`.
