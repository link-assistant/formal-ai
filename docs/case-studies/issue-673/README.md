# Issue 673: from a one-module self-AST to a workspace census

Issue [#673](https://github.com/link-assistant/formal-ai/issues/673) (E54) asks
the meta-language self-representation to stop being pinned to a single file.
Before this change, `data/meta/self-ast.lino` (R381) described exactly one
module — `src/agentic_coding/planner.rs` — so the self-coding planner (E35/E36)
could not introspect, let alone edit, anything else.

## What the census is now

`data/meta/self-ast/` holds **289 committed documents**: one per owned `src/`
module plus `index.lino`, the workspace index.

```text
self_ast_census_index
  language rust
  engine meta_language
  full_fidelity_prefix src/agentic_coding/
  module_count 288
  full_ast_module_count 43
  signature_module_count 245
  symbol_count 5928
  content_id self_ast_census_index_fe2edb27a91bfe24
  modules
    src/agent.rs signature 50 source_module_d6dc0c5386fb1128
    src/agentic_coding/capability_router.rs full_ast 10 source_module_ec1829b2308354b6
    …
```

Each module document names its target, its fidelity marker, its content id, and
every declared item with a 1-based inclusive span:

```text
self_ast_census
  target src/agentic_coding/source_links.rs
  language rust
  fidelity full_ast
  byte_len 9464
  line_count 193
  content_id source_module_6177a1573de819fc
  symbol_count 11
  symbols
    const SOURCE_LINKS_PATH 44 44
    …
  ast
    engine meta_language
    projection abstract_syntax
    text_preserved true
    …
```

### Scaling honestly

The issue explicitly asks not to seed multiple megabytes. Two tiers, each
declared in the document itself:

| Tier | Modules | What it records |
| --- | --- | --- |
| `full_ast` | 43 (`src/agentic_coding/`) | The meta-language links-network abstract-syntax census: link counts, named nodes, and a node-kind histogram, with `text_preserved` proving the parse round-trips. |
| `signature` | 245 (rest of `src/`) | Items, symbol names, kinds, and line spans from a lexical scan — enough to address any `path:symbol`, at negligible cost. |

The whole committed census is well under the 2 MB the test guards, and every
document stays under the repository's 1500-line `.lino` limit.

## Freshness is enforced, not asserted

`WorkspaceCensus::documents()` is a pure, path-sorted function of the sources, so
"what is committed" and "what the code renders" are directly comparable.
`drift_report` classifies every difference as `Missing`, `Stale`, or `Orphan`,
and three tests use it:

- `committed_census_documents_match_what_the_sources_render` compares the live
  render against the committed tree on disk (the drift-guard pattern from #559),
  and names the exact regeneration command in its failure message;
- `drift_check_fails_on_a_fixture_with_a_stale_census` proves the guard has
  teeth, on a two-module fixture perturbed three ways;
- CI runs `cargo test --test unit self_ast_census` as its own step
  (`Check self-AST census freshness`).

Regeneration is incremental by construction — `cargo run --example
regenerate_self_ast_census` rewrites only the documents whose modules changed:

```text
self-AST census: 289 documents (3 rewritten, 0 removed)
```

## Feeding E35: targets resolve through the index

`resolve_census_target` in `src/agentic_coding/general_planner.rs` routes the
planner's edit-target detection through `WorkspaceCensus::resolve`. A request
that *addresses the workspace* — a path with a directory component, or a
`path:symbol` pair — is planned against the module's real path:

| Request | Planned target |
| --- | --- |
| `In self_source_links.rs:owned_source_files, change hello to goodbye` | `src/self_source_links.rs` |
| `Replace foo with bar in self_ast_census.rs:resolve` | `src/self_ast_census.rs` |
| `замени привет на пока в файле agentic_coding/source_links.rs` | `src/agentic_coding/source_links.rs` |

A bare file name such as `main.rs` is deliberately **not** rewritten: the request
may be about the client's own working directory, not this workspace, so the
planner keeps the literal token and the pre-existing routing tests of #712 stay
green. The census fails closed on any ambiguous suffix.

`the_index_resolves_every_path_symbol_the_method_registry_knows` closes the loop
the issue asks for: all 66 methods of `MethodRegistry::from_dispatch()` have
their dispatch entry point resolved through the census index, with a real span.

## Honest notes

- The census documents are generated by a committed recipe and pinned
  byte-for-byte by a test, so the artifact's author is the code, not a
  hand-edit (CONTRIBUTING rule 1). The `.lino` shape is what
  `ModuleCensus::links_notation` renders; nothing in `data/meta/self-ast/` was
  written by hand.
- The pinned R381 artifact `data/meta/self-ast.lino` is untouched and still
  green; the workspace census lives beside it rather than replacing it.
- The full-AST tier is expensive in a debug build (~90 s for 43 modules), so it
  is computed lazily: a planner lookup pays only the ~1 s signature scan. The
  first version of this change compiled the AST eagerly and timed out the
  `issue_712_intent_routing` HTTP tests — the captured failure is in
  [`dev/log/issues/673/pulls/807/full-test-eager-ast-regression.log`](../../../dev/log/issues/673/pulls/807/full-test-eager-ast-regression.log).

## Reproduce

```bash
cargo run --example dump_self_ast_census                       # the index
cargo run --example dump_self_ast_census -- agentic_coding/source_links.rs
cargo run --example regenerate_self_ast_census                 # incremental rewrite
cargo test --test unit self_ast_census                         # all seven guards
```
