# Issue 749: general shell execution routing

## Root cause

Strict advertised-schema projection was already fixed by issue 744, but the
earlier parser still required a command to occur in a finite seed allowlist and
stopped collecting arguments at words it considered prose. Consequently unknown
binaries never routed, `run bash: echo hi` lost `hi`, and Bash/PowerShell command
lines could fall through before schema projection. OpenCode also encloses a
one-shot prompt in balanced quotes, which hid the execution prefix from the
router.

## Implementation

- Explicit multilingual run/execute prefixes establish an intent boundary and
  pass their complete remainder to the advertised shell capability. The client
  retains its existing sandbox and permission gate.
- Balanced quotes around an entire client prompt are removed before routing;
  quotes inside the command remain byte-for-byte intact.
- Required bare Git and shell-interpreter forms use a data-backed taxonomy;
  explicit passthrough covers the full POSIX/GNU, PowerShell, VCS,
  package/build, networking, and process command matrix.
- Seed-backed semantic intents cover file operations, Git, workspace-aware
  test/install/build selection, and local code search in English, Russian,
  Hindi, and Chinese. Repository scope distinguishes `rg` from web search.
- The issue 744 protocol adapter projects the planned `command` onto the exact
  advertised key (`cmd` or `command`) and supplies required fields such as
  `is_background`.

## Reproduction and verification

`tests/unit/issue_749_shell_routing.rs` reproduces argument truncation and
allowlist failures, then exercises the full command taxonomy, multilingual
prefixes, semantic intents, and deterministic Agent CLI replay. The integration
suite verifies Codex Responses `cmd`, strict shell schemas, language parity, and
the whole routing matrix over HTTP.

The real Agent CLI evidence under `agent-cli-evidence/` records a release-style
two-turn loop against `formal-ai serve`: Formal AI emitted the complete
`printf 'issue-749-agent-cli=passed\n' > issue-749-agent-cli-proof.txt` command,
the client executed it, and the generated proof file contains the marker. The
server trace records both the planned `bash` call and grounded final response.
The committed offline session is regenerated byte-for-byte by the unit test.

## External CLI matrix

The logs under `cli-matrix/` were captured through `with-formal-ai` against the
same local server build.

| Client | Advertised shell | Result |
| --- | --- | --- |
| Codex | `exec_command` (`cmd`) | Complete quoted `printf` call emitted; local `bwrap` networking setup blocked execution |
| Agent | `bash` (`command`) | Complete quoted `printf` call emitted and executed |
| OpenCode | `bash` (`command`) | Complete two-argument `echo` call emitted and executed after outer-prompt normalization |
| Gemini | none in the installed CLI mode | Server reports that a shell tool must be advertised |
| Qwen | none in the installed CLI mode | Server reports that a shell tool must be advertised |
| Grok 0.0.34 | CLI failed before its first request | Upstream `slice-ansi` startup `TypeError`, captured in `grok.log` |

Gemini and Qwen are also covered at the protocol/schema layer by the HTTP
regressions; the server deliberately does not invent tools absent from a client
request. Cursor Agent is tracked separately because its CLI does not expose a
custom base URL. The Grok startup failure is an installed-client dependency
failure, not a server routing result, and is not claimed as executed evidence.

Shell execution remains subject to the client host shell and its normal
permission/sandbox policy; this change does not grant execution privileges.
